Paginate through Pages

```graphql
type PageConnectionWithSearch {
  edges: [PageEdge]!
  nodes: [Page]!
  pageInfo: PageInfo!
  querySuggestions: JSON
  termAggregations: PageTermAggregations
}
```


### Fields

#### [`edges`](#) ([`[PageEdge]!`](/developers/demandhq-apis/demand-api/graphql/reference/objects/page-edge))




#### [`nodes`](#) ([`[Page]!`](/developers/demandhq-apis/demand-api/graphql/reference/objects/page))




#### [`pageInfo`](#) ([`PageInfo!`](/developers/demandhq-apis/demand-api/graphql/reference/objects/page-info))




#### [`querySuggestions`](#) ([`JSON`](/developers/demandhq-apis/demand-api/graphql/reference/scalars/json))

This is an object with terms in the query as keys, and a suggestion for that
term as the value. This is often referred to as 'Did you mean?'


#### [`termAggregations`](#) ([`PageTermAggregations`](/developers/demandhq-apis/demand-api/graphql/reference/objects/page-term-aggregations))

Values and the number of times they occur for the results of the query. These
will be the same for every page of a query, and should only be fetched on the first page.
