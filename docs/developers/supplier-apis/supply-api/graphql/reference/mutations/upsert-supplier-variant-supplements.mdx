Bulk upsert supplierVariantSupplements.
If you are creating supplierVariantSupplements, you must provide
SupplierVariantSupplementCreateInput instead of
SupplierVariantSupplementUpdateInput.
(These are similar, but more fields are required)

Requires suppliervariantsupplement/write permission on the requesting identity
Note that if there are Permission or UserInput Errors, the entire group will fail, not partially succeed.

```graphql
upsertSupplierVariantSupplements(
  inputs: [SupplierVariantSupplementUpdateInput!]
): [SupplierVariantSupplement!]
```


### Arguments

#### [`inputs`](#) ([`[SupplierVariantSupplementUpdateInput!]`](/developers/supplier-apis/supply-api/graphql/reference/inputs/supplier-variant-supplement-update-input))




### Type

#### [`SupplierVariantSupplement`](/developers/supplier-apis/supply-api/graphql/reference/objects/supplier-variant-supplement)

This has a one to one relationship with SupplierVariant, and provides supplemental data for the SupplierVariant.
These items get updated at a lower frequency, and are therefore separate.
